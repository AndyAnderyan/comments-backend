# Права доступу (RBAC) і гарди

Ролі: `User`, `Admin` (див. `src/users/dicts/role.enum.ts`).

Правила:
- Створення коментаря/відповіді: `User` і `Admin` (в межах доступних шарів)
- Редагування/приховування власного: Автор або `Admin`
- Видалення: лише `Admin` (каскадно по треду)
- Pin/Unpin: Автор або `Admin` за умови відсутності іншого pinned
- Перегляд прихованих: лише `Admin`
- Позначення прочитаним: будь-який користувач з доступом

Гарди бекенду (узгоджено з проектом):
- `JwtAuthGuard` — аутентифікація
- `RolesGuard(User|Admin)` — авторизація за ролями
- `CommentOwnerGuard` — автор редагує/ховає своє

Front-end UI:
- Ховати або вимикати кнопки дій згідно ролі/володіння коментарем
- Фічі для Admin (перегляд прихованих, видалення) видимі лише Admin
