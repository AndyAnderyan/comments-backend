# Критерії приймання і сценарії

## Критерії
- Користувач може створити, редагувати, приховати власний коментар
- Адміністратор має повні права (редагування, приховування, видалення, перегляд прихованих)
- Всі дії логуються (тип, дата/час, користувач, об’єкт, до/після)
- Коментарі коректно відображаються у картках об’єктів і загальному списку
- Сповіщення надходять адресатам; індикатор активний до прочитання
- Продуктивність відповідає вимогам

## Сценарії (з прикладу)
1. Створення коментаря в картці будівлі → відображення першим у списку
2. Редагування тексту + додавання адресатів → збереження і оновлення
3. Користувачі з доступом бачать як непрочитаний, можуть читати/відповісти
4. Адресати бачать індикацію у інтерфейсі → відкривають коментар
5. Адміністратор може переглядати/редагувати/приховати/видалити
6. У журналі логів є записи про створення та редагування
